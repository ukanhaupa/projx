# Frontend

React 19 + TypeScript SPA shell with auth, theming, and design tokens. Build your app surface on top.

## Quick Start

Prerequisites: Node.js 20+, pnpm 10+ (`corepack enable && corepack prepare pnpm@latest --activate`)

```bash
cp .env.example .env     # set VITE_API_URL
pnpm install
pnpm dev                 # http://localhost:5173
```

## Project Structure

```
src/
├── main.tsx              # Entry — ErrorBoundary > ThemeProvider > ToastProvider > ConfirmProvider > App
├── App.tsx               # Auth gate, Dashboard route, NotFound route
├── api.ts                # HTTP client with typed error classes (ForbiddenError, ConflictError, ValidationError, NotFoundError)
├── auth.ts               # OIDC auth (login, token storage, refresh, role helpers)
├── types.ts              # Shared types (UserInfo)
├── theme.tsx             # Light/dark theme context with localStorage persistence
├── index.css             # Design token system (70+ CSS variables)
├── components/
│   ├── Layout.tsx        # Sidebar + content shell with mobile hamburger toggle
│   ├── Toast.tsx         # Toast notification system (useToast hook; success/error/warning/info)
│   ├── ConfirmDialog.tsx # Promise-based confirm dialog (useConfirm hook; danger/primary variants)
│   ├── ErrorBoundary.tsx # React class component error boundary with "Go to Dashboard" recovery
│   └── ErrorScaffold.tsx # Standard error layout for retry / not-found / forbidden states
├── pages/
│   ├── Login.tsx         # OIDC password grant login with SVG eye toggle for password visibility
│   ├── Dashboard.tsx     # Welcome / landing page — replace with your application surface
│   └── NotFound.tsx      # Themed 404 page with link back to dashboard
```

E2E tests live at the project root: `e2e/`

## Wiring Your Own Surface

The shell ships with authentication, theming, the toast/confirm primitives, and a routed layout — everything else is yours to build:

1. Add a page under `src/pages/`.
2. Register it in `App.tsx` as a child route of the `Layout` shell:

   ```tsx
   <Route element={<Layout />}>
     <Route index element={<Dashboard />} />
     <Route path='/invoices' element={<InvoicesPage />} />
   </Route>
   ```

3. Add a nav entry in `src/components/Layout.tsx` (under the existing `Dashboard` `<NavLink>`).
4. Use `api` from `src/api.ts` for HTTP calls — it handles auth headers, token refresh, and typed errors automatically.

## Type-Safe API Calls

If your backend ships TypeScript types (generated by `projx gen entity` into `frontend/src/types/`), import them and call the API generically:

```tsx
import type { Invoice } from './types/invoice';

const items = await api.list<Invoice>('/invoices'); // typed
const item = await api.get<Invoice>('/invoices', id);
```

## Auth

`auth.ts` handles OIDC. On login, the access + refresh tokens are stored in `localStorage` and the API client adds them to every request. `App.tsx` checks `initAuth()` before mounting the routed layout.

## Theming

`theme.tsx` provides light/dark mode toggling via `useTheme()`. Tokens are defined in `index.css`. Override by adding new variables under `[data-theme="light"]` and `[data-theme="dark"]`.
